# AI 분석 기능 설계 스펙

**날짜:** 2026-04-07  
**상태:** 승인됨

---

## 개요

기존 규칙 기반 분석(수익률 숫자 임계값)을 보완하는 Claude AI 분석 기능 추가.  
사용자가 원할 때만 별도 버튼으로 실행하며, 4가지 분석 결과를 새 카드에 표시한다.

---

## 출력 항목

1. **오늘 요약** — 포트폴리오 전체 상태 한두 문장
2. **리스크 점수** — 1~10 숫자 + 이유 한 줄
3. **종목별 분석** — 각 종목에 대해 액션(매수/매도/유지) + 한줄 코멘트
4. **향후 확장 예정** — 시나리오 분석, 리밸런싱 추천, 매수 타이밍 추천 (이번 스펙 범위 외)

---

## 아키텍처

### API 키 관리
- 왼쪽 패널 하단 "AI 설정" 카드
- 사용자가 `sk-ant-...` 키 입력 → `localStorage['claude_api_key_v1']`에 저장
- 저장 성공 시 자물쇠 아이콘 → 체크 아이콘으로 전환

### AI 분석 트리거
- `render()` 완료 후 대시보드 상단에 "✨ AI 분석하기" 버튼 노출
- 키 미설정 시 버튼 클릭 → "API 키를 먼저 입력해주세요" 토스트
- 클릭 시: 버튼 비활성화 + 스피너 표시 → API 호출 → 결과 카드 렌더링

### Claude API 호출
- 모델: `claude-haiku-4-5-20251001` (빠름, 저렴)
- CORS 처리: `https://corsproxy.io/?url=https://api.anthropic.com/...` 프록시 경유
  (기존 Yahoo Finance 프록시 패턴 동일하게 적용)
- 호출 횟수: 포트폴리오 전체를 한 번에 전송 (단일 호출)

### 프롬프트 구조
포트폴리오 JSON을 시스템 컨텍스트로 전달:
```
{
  totalRate: number,       // 총 수익률 (%)
  totalValue: number,      // 총 평가금액 (원)
  stocks: [
    { name, rate, weight, value, pnl }
  ]
}
```

응답 포맷을 JSON으로 강제 (system prompt에 명시):
```json
{
  "summary": "오늘 요약 한두 문장",
  "riskScore": 7.2,
  "riskReason": "리스크 점수 근거 한 줄",
  "stocks": [
    { "name": "삼성전자", "action": "매수", "comment": "한줄 분석" }
  ]
}
```

## UI 레이아웃

### AI 설정 카드 (왼쪽 패널, 항상 표시)
```
[ AI 설정 ]
API 키  [sk-ant-••••••••••••    ] [저장]
         개인 키는 이 기기에만 저장됩니다
```

### AI 분석 버튼 (결과 상단)
```
[✨ AI 분석하기]   ← 분석 결과가 있을 때만 노출
```

### AI 분석 카드 (col-span-3, 대시보드 최상단)
```
┌─────────────────────────────────────────────┐
│ ✨ AI 분석        [2026-04-07 기준]  [새로고침]│
├──────────────────────┬──────────────────────┤
│ 📋 오늘 요약          │ 🎯 리스크 점수         │
│ "..."                │    7.2 / 10           │
│                      │ "리스크 근거..."        │
├──────────────────────┴──────────────────────┤
│ 종목별 분석                                   │
│ 삼성전자  [매수]  "..."                        │
│ 카카오    [유지]  "..."                        │
│ NVDA     [매도]  "..."                        │
└─────────────────────────────────────────────┘
```

액션 배지 색상:
- 매수 → `badge-good` (초록)
- 매도 → `badge-danger` (빨강)
- 유지 → 회색 텍스트

---

## 에러 처리

| 상황 | 처리 |
|------|------|
| API 키 없음 | 토스트: "API 키를 먼저 입력해주세요" |
| 네트워크 오류 | 토스트: "AI 분석에 실패했어요. 잠시 후 다시 시도해주세요" |
| JSON 파싱 실패 | 토스트: "응답 형식 오류. 다시 시도해주세요" |
| 프록시 오류 | 동일 네트워크 에러 처리 |

---

## 범위 외 (이번 구현 제외)

- 시나리오 분석 ("만약 삼성전자가 -10% 더 떨어지면?")
- 리밸런싱 추천 (목표 비중 설정 후 조정 금액 계산)
- 매수 타이밍 추천 (기술적 지표 기반)

위 기능은 향후 별도 스펙으로 추가 예정.
